# 本地卦象解析设计

## 目标

为天机卦增加默认启用的纯本地解卦能力。用户无需配置 API Key，也无需联网，即可依据本卦、变卦、动爻和所问事项获得结构化解读；Gemini、GLM、DeepSeek 继续作为可选增强方式。

## 知识数据

知识库以六位卦码为索引，覆盖 64 卦和 384 爻。每卦保存卦序、卦名、上下卦、卦辞原文、卦辞白话、大象、关键词、六条爻辞原文与白话，乾坤额外保存用九、用六。

- 原文校核来源：Wikisource《周易》，公版古籍，https://zh.wikisource.org/wiki/周易
- 结构化数据与白话翻译来源：koriyoshi2041/zhouyi，MIT License，https://github.com/koriyoshi2041/zhouyi
- 项目内保留来源说明与 MIT 版权声明，经典原文、来源白话和本项目生成的事项建议分别标识。

## 解析流程

问题为空时使用“综合”。非空问题通过本地关键词表匹配事业、财运、感情、健康、学业、出行；唯一命中时自动采用，零命中或多类别并列时返回待选择状态，由用户在界面选择后继续。

变爻按已确认的传统变占规则取用：零变看本卦卦辞；一变看本卦动爻；二变看本卦两动爻并以上爻为主；三变参看本卦和变卦卦辞并以本卦为主；四变看变卦两条对应静爻并以下爻为主；五变看变卦唯一对应静爻；六变时乾坤看用九、用六，其余看变卦卦辞。

解析结果固定输出卦象总览、动爻取用、变卦趋势、事项指引和免责声明。事项指引只将已选类别作为观察角度，不声称预测结果，也不替代医疗、法律或财务专业意见。

## 配置与界面

解析 Provider 增加 `local` 并作为新用户默认值。设置面板展示“本地知识库 / Gemini / GLM / DeepSeek”；本地模式隐藏 API Key，AI 模式维持原行为和旧配置兼容。点击解卦后才显示现有推演动画，本地解析不会发出网络请求。

## 边界与验证

缺失卦码、数据不完整或分类待选择都返回显式状态，不静默猜测。临时测试覆盖 64/384 完整性、分类唯一/未知/冲突、0 至 6 个动爻取用、Provider 默认与旧配置兼容、首屏渲染；验证通过后按仓库要求删除临时测试。
