# 抛币动效与摇一摇触发设计

## 目标

为现有两次抛币成卦流程增加“铜钱风暴”动效，并支持 iPhone Safari 与 Android Chrome 通过摇一摇触发抛币。按钮与传感器必须共用同一条业务流程；不支持传感器、用户拒绝授权或桌面环境下，按钮仍能完整起卦。

本次不改变当前“一次抛出三枚铜钱并生成三爻、两次形成六爻”的产品规则，也不修改本地解卦或 AI 解卦逻辑。

## 交互流程

### 启用摇一摇

- 所有移动端都通过明确的“启用摇一摇”按钮开启监听，避免页面在用户不知情时读取传感器。
- iPhone Safari 在按钮点击事件内调用 `DeviceMotionEvent.requestPermission()`；Android Chrome 在点击后直接添加监听。
- 用户拒绝后显示被拒绝状态，不自动重复申请；再次主动点击时才重新申请。
- 不支持传感器或不满足安全上下文要求时显示不可用状态，原抛币按钮保持可用。
- 监听只在尚未形成六爻且没有播放抛币动画时生效。动画期间、页面进入后台、卦象完成和组件卸载时暂停或移除监听；重新起卦后按本次会话内已启用的状态恢复。

### 触发抛币

- 按钮点击和摇一摇都调用 `triggerCoinToss(source)`，由同一处完成防重复检查、结果生成、动画启动和卦象写入。
- 触发时立即生成三枚铜钱结果并暂存，但动画落地前不展示正反面，也不写入六爻列表。
- 动画完整结束后一次性写入本次三爻。任何动画期间到达的按钮点击或传感器事件都被忽略，确保一次动作只增加三爻。
- 触发来源只用于界面状态和诊断，不影响随机结果。

## 组件与职责

### `App.tsx`

- 计算 `canToss`，统一管理本次结果、抛币状态和六爻写入。
- 提供唯一的 `triggerCoinToss(source)` 入口给按钮与摇一摇回调。
- 管理是否静音，并将偏好保存在 `localStorage`。
- 不直接处理设备事件、加速度计算或具体动画样式。

### `hooks/useShakeToToss.ts`

- 管理 `unsupported`、`needs-permission`、`enabled`、`denied`、`error` 五种状态。
- 封装 iOS 授权差异、事件监听与清理、页面可见性变化和冷却期。
- 仅在 `enabled && canToss` 时把有效摇动转换为一次 `onShake()` 调用。
- 通过局部类型守卫描述带 `requestPermission` 的 iOS 事件构造器，不在业务代码中散布 `any`。

### `services/shakeDetection.ts`

- 保存与 React、浏览器 API 无关的摇动检测状态，便于使用模拟采样验证。
- 优先使用已去除重力的 `event.acceleration`；只有 `accelerationIncludingGravity` 时，计算相邻采样向量差，过滤静态重力与缓慢倾斜。
- 去除重力的加速度向量阈值初始设为 `13m/s²`；回退到相邻采样差值时阈值设为 `12m/s²`。阈值集中为命名常量，设备验证后只能基于实测统一调整，不能分散硬编码。
- 在 `650ms` 窗口内出现两次间隔至少 `80ms` 的有效峰值才触发，触发后进入 `2s` 冷却期。
- 缺失坐标、非有限数值和低频缓慢变化都不计为有效峰值。

### `components/CoinTossStage.tsx`

- 负责三枚铜钱、轨迹、火花、结果标签和静音按钮的整体场景，使用稳定尺寸避免动画改变布局。
- 接收抛币阶段、暂存结果和静音状态，不读取传感器，也不生成业务结果。
- 三枚铜钱使用不同 CSS 变量控制延迟、横向偏移和旋转角度，保持错峰但共享同一时间线。

### `components/Coin.tsx`

- 收敛为单枚铜钱的纯视觉组件，只根据正反面、索引和动画阶段渲染。
- 移除现有的设备方向监听、权限请求和独立翻转状态，避免三枚铜钱重复申请权限和重复订阅事件。

### `services/coinFeedback.ts`

- 使用 Web Audio 在本地合成起飞轻响与落地金属碰撞声，不增加音频文件或网络请求。
- 音频上下文只在用户完成过点击交互后创建或恢复，浏览器阻止自动播放时静默降级。
- 落地时在支持的平台调用短促 `navigator.vibrate`；不支持震动的平台直接跳过。

## 铜钱风暴时间线

- 抛币区保持固定高度和响应式宽度，移动端三枚铜钱不会超出容器或互相遮挡。
- 蓄力 `180ms`：铜钱轻微下沉，背景亮度收紧。
- 翻飞 `900ms`：三枚铜钱错峰执行抛物线位移、立体翻面和横向偏移，同时显示短促金色轨迹与少量火花。
- 落定 `420ms`：铜钱依次回弹、显示最终正反面，并播放落地音效与可用设备震动。
- 总时长 `1.5s`，以单一阶段状态和 CSS 动画完成，不引入 Canvas 或第三方动画库。
- 开启 `prefers-reduced-motion` 时取消高速旋转、轨迹和火花，只保留短淡入与结果落定。

## 状态反馈与无障碍

- 摇一摇启用按钮、静音按钮使用 Lucide 图标，并提供可见命令文本或明确的 `aria-label` 与 `title`。
- 授权中、已启用、被拒绝、不可用和检测到摇动使用简短状态文本；动态状态通过 `aria-live="polite"` 输出。
- 抛币动画容器在进行中设置 `aria-busy="true"`，最终结果仍以文字展示，动画不作为唯一信息来源。
- 页面隐藏时不保留活跃传感器监听或计时器，防止后台误触发和资源泄漏。

## 错误与降级

- 传感器权限异常、事件数据异常、Web Audio 创建失败和震动不可用都不能阻断按钮抛币。
- 授权错误只更新传感器状态并记录必要日志，不使用阻塞式弹窗。
- 传感器需要 HTTPS 安全上下文；本地 `localhost` 可直接调试，部署环境未使用 HTTPS 时按不支持处理。
- 重置起卦时取消尚未结束的动画计时器和暂存结果，避免旧动画向新卦写入数据。

## 验证

按仓库规则使用临时测试文件，验证通过后删除测试文件和空目录，不提交测试代码。

- 纯函数测试：静止、缓慢倾斜、单峰、双峰正常摇动、冷却期连续强摇、无效采样。
- 组件与流程测试：按钮和摇一摇走同一入口；一次动作只增加三爻；两次形成六爻；动画期间不重复触发；重置后旧结果不回写。
- 权限与降级测试：iOS 授权允许、拒绝和异常；Android 无授权函数；无设备 API；声音被浏览器阻止；震动 API 缺失。
- 静态检查：TypeScript、生产构建与 `git diff --check`。
- 设备验证：iPhone Safari 和 Android Chrome 各验证一次授权、摇动阈值、误触发、两次起卦、静音和按钮降级。当前自动化环境无法模拟真实传感器，因此设备验证结果必须单独记录，不能用桌面模拟结果代替。

## 不引入的内容

- 不增加第三方动画、物理或手势依赖。
- 不把设备姿态用于持续控制铜钱位置，避免眩晕和误操作。
- 不改变起卦概率、本地知识库、AI Provider 或解析动画。
